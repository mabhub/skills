# Checklist de révision éditoriale

Utiliser cette checklist pour vérifier la qualité d'une documentation technique avant publication.

## 1. Corrections grammaticales et linguistiques

- [ ] Aucune faute d'orthographe
- [ ] Aucune erreur de grammaire ou syntaxe
- [ ] Accords corrects (genre, nombre, conjugaisons)
- [ ] Ponctuation appropriée
- [ ] Pas d'anglicismes inappropriés
- [ ] Phrases fluides et lisibles

## 2. Structure et cohérence

- [ ] Hiérarchie de titres logique (H1 > H2 > H3)
- [ ] Progression logique de l'information
- [ ] Sections thématiques bien définies
- [ ] Titres et sous-titres uniformes
- [ ] Transitions fluides entre sections
- [ ] Table des matières si document long (>5 sections)

## 3. Ton et accessibilité

- [ ] Ton professionnel et cohérent
- [ ] Langage accessible au public cible
- [ ] Verbes d'action privilégiés
- [ ] Phrases courtes (<25 mots)
- [ ] Voix active privilégiée sur passive
- [ ] Jargon expliqué ou évité

## 4. Formatage et présentation

- [ ] **Gras** utilisé pour concepts clés
- [ ] `Code inline` pour noms techniques
- [ ] Blocs de code avec syntaxe colorée
- [ ] Listes à puces bien structurées
- [ ] Tableaux pour données tabulaires
- [ ] Citations/notes pour informations importantes
- [ ] Lignes limitées à ~100 caractères

## 5. Enrichissement du contenu

- [ ] Introduction expliquant l'importance/utilité
- [ ] Cas d'usage pratiques fournis
- [ ] Exemples concrets et fonctionnels
- [ ] Bénéfices utilisateur explicites
- [ ] Recommandations et bonnes pratiques
- [ ] Mises en garde sur pièges courants

## 6. Exactitude technique

- [ ] Informations techniques correctes
- [ ] Exemples de code fonctionnels
- [ ] Liens valides et à jour
- [ ] Références précises
- [ ] Versions de technologies mentionnées
- [ ] Screenshots/diagrammes à jour

## 7. Préservation (pour révisions)

- [ ] Structure MDX/composants préservée
- [ ] Tous les liens maintenus
- [ ] Composants interactifs intacts
- [ ] Sens technique non modifié
- [ ] Métadonnées conservées

## 8. Cohérence globale

- [ ] Terminologie uniforme dans tout le document
- [ ] Style de code cohérent
- [ ] Conventions de nommage respectées
- [ ] Format des exemples uniforme
- [ ] Même niveau de détail par section

## 9. Accessibilité web (si applicable)

- [ ] Texte alternatif pour images
- [ ] Contraste suffisant
- [ ] Hiérarchie sémantique HTML
- [ ] Liens descriptifs
- [ ] Tableaux avec en-têtes

## 10. Publication

- [ ] Titre descriptif et explicite
- [ ] Métadonnées complètes (auteur, date, version)
- [ ] Tags/catégories appropriés
- [ ] Changelog si document versionné
- [ ] Licence mentionnée si nécessaire

---

## Validation finale

Avant de marquer le document comme terminé :

1. **Relecture complète** : Lire du début à la fin
2. **Test des exemples** : Vérifier que le code fonctionne
3. **Vérification des liens** : Tester tous les liens
4. **Peer review** : Faire relire par un collègue
5. **Test utilisateur** : Demander un retour au public cible

## Niveaux de qualité

### ⭐ Niveau minimum (publication acceptable)

- Pas d'erreurs de grammaire/orthographe
- Structure logique
- Informations techniques correctes

### ⭐⭐ Niveau standard (qualité professionnelle)

- Checklist complète validée
- Exemples pratiques inclus
- Formatage cohérent

### ⭐⭐⭐ Niveau excellence (documentation exemplaire)

- Enrichissement de contenu optimal
- Cas d'usage variés
- Visuels/diagrammes explicatifs
- Ressources complémentaires
