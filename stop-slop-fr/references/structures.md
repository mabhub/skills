# Structures à éviter

## Contrastes binaires

Créent un faux suspense. Énoncer le point directement.

| Pattern | Problème |
|---------|----------|
| "Ce n'est pas X. C'est Y." | Retournement télégraphié |
| "Il ne s'agit pas de X mais de Y" | Reformulation formulaïque |
| "La question n'est pas X. Elle est Y." | Faux recadrage |
| "Le problème n'est pas X. C'est Y." | Même formule |
| "Ce n'est pas tant X que Y" | Nuance factice |
| "Loin de X, il s'agit plutôt de Y" | Mise en scène de la nuance |
| "Au-delà de X, c'est Y qui..." | Fausse profondeur |
| "Non pas X, mais bien Y" | Correction théâtrale |
| "Moins X que Y" | Comparaison creuse |

**Correction :** énoncer Y directement. "Le problème est Y." Supprimer la négation.

## Listes négatives

Lister ce que quelque chose n'est *pas* avant de révéler ce qu'elle *est*. Un effeuillage rhétorique.

| Pattern | Problème |
|---------|----------|
| "Ce n'est pas un X... Ni un Y... C'est un Z." | Suspense artificiel par négation |
| "Ce n'était pas X. Ce n'était pas Y. C'était Z." | Même structure, passé |
| "Il ne s'agit ni de X, ni de Y, mais de Z" | Variante académique |

**Correction :** énoncer Z. Le lecteur n'a pas besoin de la piste d'élan.

## Constructions clivées excessives

Le français IA adore les constructions emphatiques "c'est... qui/que". Elles sont idiomatiques en français — le problème est leur accumulation.

| Pattern | Problème |
|---------|----------|
| "C'est là que réside tout l'enjeu" | Fausse emphase |
| "C'est précisément ce qui fait que" | Précision de façade |
| "C'est à ce niveau que se joue" | Spatialisation abstraite |
| "C'est bien là tout le paradoxe" | Paradoxe annoncé, jamais montré |
| "C'est ce qui rend la chose" | Annonce vide |
| "C'est dans cette perspective que" | Cadrage inutile |

**Correction :** syntaxe directe. "L'enjeu est X" plutôt que "C'est là que réside l'enjeu."

## Tournures impersonnelles en cascade

Le tic français IA par excellence. Les LLM construisent des phrases entières sur des "il" impersonnels pour éviter de nommer un sujet.

| Pattern | Correction |
|---------|------------|
| "Il est crucial de comprendre que" | (Énoncer le fait) |
| "Il s'avère que" | (Énoncer le fait) |
| "Il apparaît que" | (Énoncer le fait) |
| "Il s'agit de" | (Nommer le sujet) |
| "Il en résulte que" | (Conséquence directe) |
| "Il importe de souligner" | (Énoncer le point) |
| "Il en va de même pour" | "X aussi" / "De même, X" |
| "Il est à noter que" | (Énoncer le fait) |
| "Il reste que" | (Énoncer le fait) |

**Règle :** si une phrase commence par "il" impersonnel suivi d'un verbe + "que/de", chercher le vrai sujet et restructurer.

## Nominalisation excessive

Les LLM transforment les verbes en noms abstraits, vidant les phrases de leur énergie. Problème spécifiquement français.

| Nominalisation | Version directe |
|----------------|-----------------|
| "La mise en place de" | Installer, créer, déployer |
| "La réalisation de" | Faire, produire (ou le verbe précis) |
| "L'optimisation de" | Améliorer |
| "La prise en compte de" | Considérer, intégrer |
| "L'accompagnement de" | Aider, former (ou l'action précise) |
| "Le déploiement de" | Déployer |
| "La mise en œuvre de" | Appliquer, faire |
| "L'appréhension de" | Comprendre |
| "La finalisation de" | Finir, terminer |
| "La consolidation de" | Renforcer, stabiliser |
| "L'identification de" | Trouver, repérer |
| "La sécurisation de" | Sécuriser, protéger |

**Règle :** si une phrase contient "[Article] + [nom en -tion/-ment/-age] + de", vérifier s'il existe un verbe plus direct.

## Fausse agence

Les objets inanimés n'agissent pas. Quelqu'un agit sur eux.

| Pattern | Problème |
|---------|----------|
| "Le projet porte une ambition" | Le projet ne porte rien. L'équipe ambitionne. |
| "Cette démarche ouvre des perspectives" | Quelqu'un voit des perspectives. |
| "L'analyse révèle que" | Quelqu'un analyse et constate. |
| "Le texte suggère que" | L'auteur suggère. |
| "Le contexte invite à" | Les contextes n'invitent pas. Des gens font des choix. |
| "La question se pose de savoir si" | Qui se pose cette question ? |
| "Le constat s'impose" | Quelqu'un constate. |
| "La tendance se confirme" | Des données montrent que / des gens observent que. |
| "Le rapport met en lumière" | Les auteurs du rapport montrent que. |
| "La situation exige" | (Qui exige ?) |

**Correction :** nommer l'humain. "L'équipe a constaté que" bat "Le constat s'impose que." Si aucune personne précise ne convient, utiliser "vous" ou "on" (sujet réel).

## Voix passive évasive

Le français utilise plus naturellement la voix passive que l'anglais. Ne pas bannir systématiquement. Cibler la voix passive qui masque l'acteur quand l'acteur est important.

| Acceptable | À revoir |
|-----------|----------|
| "La loi a été votée en 2024" (l'Assemblée est implicite) | "Des erreurs ont été commises" (par qui ?) |
| "Le bâtiment a été construit au XIXe siècle" (informatif) | "La décision a été prise" (par qui ?) |
| "Le rapport est publié chaque année" (processus) | "Il a été décidé que" (qui a décidé ?) |

Aussi cibler la tournure pronominale évasive :

- "La question se pose" → qui la pose ?
- "Le problème se résout" → qui le résout ?
- "L'idée se répand" → qui la diffuse ?

## Listes à trois éléments

Les LLM adorent les triplets. Deux éléments ou un seul suffisent.

| Pattern | Correction |
|---------|------------|
| "clarté, précision et rigueur" | Garder deux éléments ou un seul |
| "innovant, performant et durable" | Choisir le plus pertinent |
| "comprendre, analyser et agir" | "Comprendre et agir" |
| "simple, efficace et fiable" | "Simple et fiable" |

## Fausses questions rhétoriques

| Pattern | Problème |
|---------|----------|
| "Mais alors, comment expliquer que... ?" | Questionnement socratique de façade |
| "Quelle leçon en tirer ?" | Condescendant |
| "Faut-il pour autant... ?" | Faux débat |
| "Et si la solution était... ?" | Révélation télégraphiée |
| "Comment dès lors concilier... ?" | Fausse complexité |
| "Peut-on vraiment parler de... ?" | Question dont la réponse suit immédiatement |

**Correction :** énoncer le point directement. Le lecteur tire ses propres conclusions.

## Conclusions génériques positives

| Pattern | Problème |
|---------|----------|
| "Les perspectives sont prometteuses" | Vide |
| "L'avenir nous le dira" | Cliché |
| "Le chemin reste long mais les bases sont posées" | Faux équilibre |
| "Cette évolution n'en est qu'à ses débuts" | Non-information |
| "Autant de défis que d'opportunités" | Formule creuse |
| "Un pas dans la bonne direction" | Évaluation sans contenu |
| "Reste à voir si..." | Fausse ouverture |

## Patterns de rythme

| Pattern | Correction |
|---------|------------|
| Trois phrases consécutives de même longueur | En casser une |
| Questions immédiatement répondues | Laisser la question respirer ou la couper |
| Chaque paragraphe finit par une phrase-choc | Varier les fins |
| Plus d'un tiret cadratin par paragraphe | Réduire — l'abus crée une fausse complexité |
| Empilement de phrases courtes percutantes | Ne pas empiler les fragments |
